# Grill the PM

Run a `/grilling` session, using the `/sharpen-domain-language` skill.

The person you're interviewing is a **product owner, not the technical architect.** Drive hard on terminology precision, the domain glossary, and behavioral scenarios — but do **not** ask them to make architecture or technology trade-offs, and do **not** record ADRs or cross-examine source code. If such a decision surfaces, name it in one line, note that it belongs to technical design, and move on.
